# Code of Conduct

This project is a professional engineering space. Be direct, specific, and
respectful.

## Expected Behavior

- Discuss code, designs, tradeoffs, and evidence rather than personal motives.
- Assume security reports and data-loss reports may be sensitive.
- Redact secrets, tokens, account identifiers, OAuth capture blobs, and local
  credential files before posting.
- Respect maintainers' requests to move vulnerability details to a private
  security advisory.

## Unacceptable Behavior

- Harassment, personal attacks, or discriminatory language.
- Posting another person's private data, credentials, tokens, or local app state.
- Pushing users to disclose secrets or account details in public issues.
- Repeatedly derailing technical discussions after a maintainer asks to refocus.

## Enforcement

Maintainers may edit or remove comments, lock discussions, close issues, or
restrict participation when needed to keep the project safe and focused.

Security vulnerabilities should be reported through the repository security
advisory flow instead of public issues.
